Markdown in Activity Descriptions

If you’ve ever wanted to keep useful activity notes in Strava, but were not excited about the lack of formatting in the description field, now you have options. While A Race athlete Event Descriptions have had Markdown support since day one, we have just added support for Strava Activity Description.

In Strava, this field is found just under the Title field on the Edit activity page. Descriptions are public so keep in mind you are sharing this information with the Strava community.

When you complete a activity, just type in your notes, people you met, whatever. It might look something like this on Strava

Then when you go to A Race athlete, it will be formatted and look like this.

Note: If you’ve updated your activity since your first upload and don’t see your changes, you may need to press the activity refresh button to sync it with Strava.

Markdown has become popular as a text markup language for sites like GitHub because it is light weight and easy to read and write as plain text. In fact, this blog post is written in Markdown. If you are not familiar with Markdown, don’t worry, you can use as much or as little as you want. Here are some of the basics:

Note: Line spaces are important. Two adjacent lines of markdown will be combined as a single paragraph. If you aren’t seeing the results you expect, make sure you have line returns in the right place.

Markdown Basics

Hash (pound) marks followed by a space and text creates the headers above. Headings range from 1 - 6 and coincide with html h1 - h6 tags.

### Three
###### Six

Emphasis

Everyone loves to use bold and italics, rebels use bold italics together.

Everyone loves to use **bold** and *italics*, rebels use _**bold italics**_ together.

Lists

Unordered

Make sure you remember to leave a line between your preceding text and your first list item. Note that there are three spaces before the asterisks on the indented items.

  • Helmet
  • Shoes

  • Bike

    • Wheels
    • Cassette
* Helmet
* Shoes
* Bike
   * Wheels
   * Cassette

Ordered

Starting a line with a number and dot is all you need. I’m using all 1. to show that these will be converted to their proper order when rendered. Note that there are three spaces before the asterisks on the indented items.

  1. Helmet
  2. Shoes

  3. Bike

    1. Wheels
    2. Cassette
1. Helmet
1. Shoes
1. Bike
   1. Wheels
   1. Cassette
